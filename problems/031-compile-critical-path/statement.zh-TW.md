# 編譯管線的關鍵路徑

一個專案的編譯工作由 `N` 個 stage 組成。第 `i` 個 stage 需要 `d_i` 單位時間。
若有一條相依關係 `u v`，stage `v` 必須等 stage `u` 完成後才能開始。

相依關係保證形成有向無環圖。系統擁有無限多個 Worker，因此所有相依條件
已滿足的 stage 都能同時開始。沒有前驅的 stage 在時間 `0` 開始。

一條完整管線是從入度為零的 stage 走到出度為零的 stage 的有向路徑。
專案的最早完成時間等於完整管線的最大 stage 時間總和。請輸出最早完成時間，
以及達到這個時間的完整管線數量。管線數量可能很大，請對
`1,000,000,007` 取餘數。

只有經過的 stage 序列完全相同，兩條管線才視為同一條。孤立 stage 同時是
source 與 sink，因此自己構成一條完整管線。

## 輸入

第一行包含兩個整數 `N M`。

第二行包含 `N` 個整數 `d_1 ... d_N`。

接下來 `M` 行各包含兩個整數 `u v`，表示一條由 `u` 指向 `v` 的相依關係。
不會有重複邊或 self-loop。

## 輸出

輸出兩個整數：最早完成時間，以及達到該時間的完整管線數量模
`1,000,000,007`。

## 限制

- `1 <= N <= 200,000`
- `0 <= M <= 400,000`
- `1 <= d_i <= 10^9`
- 所有 stage 時間總和不超過 `9 × 10^18`
- 輸入圖是 DAG
- stage ID 採 1-based 編號
